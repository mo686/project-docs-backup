# 工装测试上位机自动化联动方案

## 一、现状分析

### 1.1 当前系统架构

系统由两个独立的上位机组成，分别承担不同职责：

| 上位机 | 功能 | 通信协议 | 硬件模块 |
|--------|------|----------|----------|
| `1.py` | 18路电压采集 + 告警 | Modbus RTU (pymodbus) | 电压采集模块 |
| `NTC.py` | 6通道NTC温度模拟输出 | Modbus RTU (minimalmodbus) | 模拟输出模块 |

### 1.2 当前工作流程

```
┌─────────────────┐         ┌──────────────────┐
│    NTC.py        │         │      1.py          │
│  (温度模拟输出)   │         │   (电压采集监测)    │
│                  │         │                    │
│  用户设温 → 电压  │  写入   │  读到电压 → 判阈值  │
│  → Modbus写入    │────────→│  → 告警/显示       │
│       ↓          │  模块   │                    │
│  硬件DAC输出      │         │  硬件ADC采集        │
│  → 主控板ADC引脚  │         │  ← 18路测试点       │
└─────────────────┘         └──────────────────┘
```

**核心问题**：两个上位机完全独立运行，NTC.py 输出的电压变化需要人工观察 1.py 的回读数据后才能手动调整，无法形成闭环自动化。

### 1.3 工装测试的真实场景


## 二、联动自动化目标

### 2.1 核心目标：温度-电压闭环自动化

```
NTC.py设定目标温度
       │
       ▼
NTCConverter.temp2voltage(t)  ← 计算期望电压
       │
       ▼
HKModule.write_voltage(ch, v)  ← 输出至硬件模块
       │
       ▼
硬件模块 DAC → 被测主控板 ADC 引脚 → 实际分压电路
       │
       ▼ (链路回环)
       │
1.py 读到实际电压 ← ModbusRTUClient.read_voltage_data()
       │
       ▼
联动引擎对比期望电压 vs 实际电压
       │
       ├── 偏差 ≤ 容差 → 稳定，继续
       │
       └── 偏差 > 容差 → 自动补偿调整
              ├── 调整NTC.py输出补偿量
              └── 异常大偏差 → 告警(测试夹具接触不良/板卡损坏)
```

### 2.2 具体功能目标

| 编号 | 功能目标 | 描述 |
|------|----------|------|
| F1 | **通道自动匹配** | 用户选择关联通道对(NTC输出CHx ↔ 1.py采集CHy)，一键绑定 |
| F2 | **预设斜率自动调节** | 自动模式下依据预设斜率升温/降温，实时对比回读值 |
| F3 | **电压超标自动限幅** | 某通道回读电压超阈值 → 自动降低该通道斜率或暂停 |
| F4 | **通道间联动** | 通道A电压超标 → 联动降低通道B斜率(如大电流驱动关联) |
| F5 | **自动补偿校准** | 长期运行发现输出-回读偏差 → 自动补偿修正 |
| F6 | **一键全自动测试** | 设置测试参数 → 启动 → 全程无人值守，异常自动记录 |

## 三、技术方案设计

### 3.1 总体方案：新增联动调度引擎

采用**观察者模式 + 共享数据库**架构，最小化对现有代码的侵入：

```
┌─────────────────────────────────────────────────────────────────┐
│                    AutoCoordinator 联动调度引擎                   │
│                                                                 │
│  输入：                                                          │
│  ├── NTC.py 当前输出状态（温度/电压/曲线/斜率）                   │
│  ├── 1.py 实时采集数据（18路电压）                                │
│  └── 用户配置（通道绑定表、斜率参数、限制值、容差）                │
│                                                                 │
│  输出(控制)：                                                     │
│  └── → NTC.py 调整温度设定值、斜率、启停                          │
│                                                                 │
│  定时循环 (100ms)                                                │
│  ┌──────────────────────────────────────────────────────┐       │
│  │  1. 读取1.py最新18路电压数据                          │       │
│  │  2. 读取NTC.py当前6通道温度/电压/斜率状态              │       │
│  │  3. 对每个关联通道:                                   │       │
│  │     actual = 1.py采集电压[关联CH]                     │       │
│  │     expected = NTCConverter.temp2voltage(NTC当前温度)  │       │
│  │     偏差 = actual - expected                          │       │
│  │     if |偏差| > 容差:                                  │       │
│  │       → 自动补偿或告警                                │       │
│  │  4. 检查电压是否超阈值                                │       │
│  │     超阈值 → 自动调整该通道/关联通道斜率              │       │
│  │  5. 自动模式温度推进                                  │       │
│  │     按斜率计算下一步温度 → 限温裁剪 → 写入NTC.py      │       │
│  │  6. 记录联动日志                                      │       │
│  └──────────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────────┘
```

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────┐
│ 写入状态     │────→│  NTC.json        │────→│ 读取状态     │
│ (温度,电压,  │     │  (NTC.py配置文件) │     │              │
│  斜率,运行)  │     └──────────────────┘     │              │
└──────────────┘                              └──────────────┘

│  └ link_status: 联动状态表(新增)              │  │

      ↑ 读取                                       │
      │                                            │
┌──────────────┐     ┌──────────────────┐        │
```

### 3.4 联动引擎核心代码设计

```python
# auto_coordinator.py — 联动调度引擎
import json
import time
import sqlite3
import logging
from pathlib import Path
from threading import Thread, Lock

class AutoCoordinator:
    """
    联动调度引擎
    职责：桥接 1.py(电压采集) 与 NTC.py(温度模拟输出)，
         实现温度-电压闭环自动控制
    """
    
    def __init__(self, db_path, ntc_json_path):
        self.db_path = db_path
        self.ntc_json_path = Path(ntc_json_path)
        self.running = False
        self.lock = Lock()
        
        # 通道绑定配置: {ntc_ch: voltage_ch, ...}
        self.mapping = {}
        # 容差配置
        self.tolerances = {}
        # 自动补偿开关
        self.compensation_enabled = {}
        
        # 控制参数
        self.scan_interval = 100  # ms
        
    def load_mapping(self):
        """从数据库加载通道绑定配置"""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        cursor.execute("SELECT ntc_channel, voltage_channel, tolerance, compensation_enabled FROM channel_mapping")
        for row in cursor.fetchall():
            ntc_ch, v_ch, tol, comp = row
            self.mapping[ntc_ch] = v_ch
            self.tolerances[ntc_ch] = tol
            self.compensation_enabled[ntc_ch] = bool(comp)
        conn.close()
    
    def read_ntc_state(self):
        """读取 NTC.py 当前状态 (通过 NTC.json)"""
        try:
            with open(self.ntc_json_path, 'r', encoding='utf-8') as f:
                cfg = json.load(f)
            return {
                'temps': cfg.get('ntc_temps', [25]*6),
                'curves': cfg.get('ntc_curves', ['10k']*6),
                'slopes': cfg.get('slope', [5.0]*6),
                'auto_run': cfg.get('auto_run', [False]*6),
                'limit_high': cfg.get('limit_temp_high', [100]*6),
                'limit_low': cfg.get('limit_temp_low', [-50]*6),
                'pullup_resistors': cfg.get('pullup_resistors', [10.0]*6),
            }
        except Exception as e:
            logging.error(f"读取NTC.json失败: {e}")
            return None
    
    def write_ntc_command(self, channel, temp=None, slope=None, auto_run=None):
        """
        向 NTC.py 发送控制指令 (通过修改 NTC.json)
        然后 NTC.py 的定时器轮询检测到变更后应用
        """
        # 注：实际实现需配合 NTC.py 增加 JSON 轮询监听
        pass
    
    def read_voltage_data(self):
        """从数据库读取 1.py 最新采集的电压数据"""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        cursor.execute(
            "SELECT channel1,channel2,channel3,channel4,channel5,channel6,"
            "channel7,channel8,channel9,channel10,channel11,channel12,"
            "channel13,channel14,channel15,channel16,channel17,channel18 "
            "FROM voltage_data ORDER BY id DESC LIMIT 1"
        )
        row = cursor.fetchone()
        conn.close()
        if row:
            return list(row)
        return [0.0] * 18
    
    def control_loop(self):
        """主控制循环"""
        while self.running:
            with self.lock:
                try:
                    # 1. 获取两端状态
                    voltages = self.read_voltage_data()
                    ntc_state = self.read_ntc_state()
                    if not ntc_state:
                        time.sleep(self.scan_interval / 1000)
                        continue
                    
                    # 2. 逐通道检查
                    for ntc_ch, v_ch in self.mapping.items():
                        idx = ntc_ch - 1  # 0-based
                        
                        actual_v = voltages[v_ch - 1]
                        expected_v = self._calc_expected_voltage(
                            ntc_state['temps'][idx],
                            ntc_state['curves'][idx],
                            ntc_state['pullup_resistors'][idx]
                        )
                        deviation = actual_v - expected_v
                        tol = self.tolerances.get(ntc_ch, 0.05)
                        
                        # 3. 偏差处理
                        if abs(deviation) > tol:
                            self._handle_deviation(ntc_ch, deviation, actual_v, expected_v)
                        
                        # 4. 超阈值检查(如果电压超过1.py设置的阈值)
                        if actual_v > self._get_threshold(v_ch):
                            self._handle_over_threshold(ntc_ch, v_ch, actual_v)
                        
                        # 5. 自动斜坡推进(如果启用自动模式)
                        if ntc_state['auto_run'][idx]:
                            self._handle_auto_ramp(ntc_ch, ntc_state, idx)
                    
                    # 6. 写入联动状态日志
                    self._log_link_state()
                    
                except Exception as e:
                    logging.error(f"联动控制循环异常: {e}")
            
            time.sleep(self.scan_interval / 1000)
    
    def _handle_deviation(self, ntc_ch, deviation, actual_v, expected_v):
        """处理输出-回读偏差"""
        logging.warning(
            f"NTC CH{ntc_ch}: 偏差={deviation:.3f}V, "
            f"期望={expected_v:.3f}V, 实际={actual_v:.3f}V"
        )
        
        if abs(deviation) > 0.5:  # 大偏差 → 硬件故障
            self._trigger_alarm(ntc_ch, f"大偏差: {deviation:.3f}V, 检查夹具接触/板卡状态")
        elif self.compensation_enabled.get(ntc_ch, True):
            # 小偏差 → 自动补偿
            compensation = -deviation * 0.5  # 50%补偿避免过冲
            self._log_event('COMPENSATION', f"NTC CH{ntc_ch} 自动补偿 {compensation:.3f}V")
    
    def _handle_over_threshold(self, ntc_ch, v_ch, actual_v):
        """处理电压超标"""
        threshold = self._get_threshold(v_ch)
        logging.warning(f"NTC CH{ntc_ch} → 1.py CH{v_ch}: {actual_v:.3f}V 超过阈值 {threshold:.3f}V")
        
        # 自动调整该通道斜率(减半) + 联动关联通道
        self._adjust_slope(ntc_ch, -0.5)  # 斜率减半
        
        # 检查通道间联动配置
        for other_ntc, other_v in self.mapping.items():
            if other_ntc != ntc_ch and abs(actual_v - self._get_threshold(v_ch)) < 0.2:
                self._adjust_slope(other_ntc, -0.3)  # 联动通道也减30%
                self._log_event('LINK_TRIGGER', 
                    f"NTC CH{ntc_ch}超限 → 联动降低 NTC CH{other_ntc} 斜率")
    
    def _handle_auto_ramp(self, ntc_ch, ntc_state, idx):
        """自动斜坡推进"""
        slope = ntc_state['slopes'][idx]  # °C/min
        current_temp = ntc_state['temps'][idx]
        limit_high = ntc_state['limit_high'][idx]
        limit_low = ntc_state['limit_low'][idx]
        
        delta = (slope / 60.0) * (self.scan_interval / 1000.0)
        new_temp = current_temp + delta
        
        if slope >= 0 and new_temp >= limit_high:
            new_temp = limit_high
            self._log_event('LIMIT_HIT', f"NTC CH{ntc_ch} 到达上限 {limit_high}°C")
        elif slope < 0 and new_temp <= limit_low:
            new_temp = limit_low
            self._log_event('LIMIT_HIT', f"NTC CH{ntc_ch} 到达下限 {limit_low}°C")
        
        self.write_ntc_command(ntc_ch, temp=new_temp)
    
    def _adjot_slope(self, ch, multiplier):
        """调整通道斜率（乘以系数）"""
        pass  # 通过修改 NTC.json 实现
    
    def start(self):
        """启动联动引擎"""
        self.load_mapping()
        self.running = True
        Thread(target=self.control_loop, daemon=True).start()
        logging.info("联动调度引擎已启动")
    
    def stop(self):
        """停止联动引擎"""
        self.running = False
        logging.info("联动调度引擎已停止")
```

## 六、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| JSON文件跨进程写入竞争 | 配置丢失或读脏数据 | 使用临时文件+原子重命名 |
| 数据库并发写入冲突 | 数据不完整 | SQLite WAL模式天然支持一写多读 |
| 控制指令延迟过大 | 无法及时响应 | 监控延迟指标，>500ms告警 |
| 自动补偿导致振荡 | 输出不稳定 | 限制补偿步长+死区控制 |
| NTC.py异常退出 | 测试中断 | 看门狗机制，检测进程存活 |
