# 联动调度引擎 使用说明

## 1. 项目概述

本系统是一个**家电产线测试自动化平台**，由三个组件协同工作：

| 组件 | 功能 | 启动方式 |
|------|------|----------|
| **电压采集上位机** (1.py) | 18通道 Modbus RTU 电压采集、波形显示、数据存储 | 独立启动 `python 1.py` |
| **NTC 模拟器** (NTC.py / NTCSimTab) | 6通道 NTC 热敏电阻温度-电压模拟、曲线加载 | 可独立或集成启动 |
| **联动调度引擎** (auto_coordinator) | 自动联调：阈值触发 NTC 温度斜坡，电压超标时自动升温，恢复后停止 | `run_coordinator.bat` 或 `python -m auto_coordinator.app` |

系统工作流程：电压采集上位机实时采集 18 路电压 → 联动引擎监测电压是否超阈值 → 超阈值时触发对应 NTC 通道按预设斜率升温 → 电压回落后自动停止升温。

控制策略采用**开环阈值触发模式**：电压超过设定阈值时，绑定 NTC 通道启动温度斜坡；电压降至阈值以下后，停止斜坡。无需闭环反馈补偿。

## 2. 环境要求

- **Python** 3.8+
- **依赖包**：
  ```
  PyQt5
  pyqtgraph
  pymodbus (>=3.0)
  pandas
  openpyxl
  numpy
  minimalmodbus
  pyserial
  ```
  安装命令：
  ```bash
  pip install PyQt5 pyqtgraph pymodbus pandas openpyxl numpy minimalmodbus pyserial
  ```

- **硬件**：支持 Modbus RTU 协议的电压采集模块、HK 系列串口输出模块
- **数据库**：使用 SQLite3（Python 内置），无需额外安装

## 4. 详细功能说明

### 4.1 电压采集（第二个标签页）

#### 功能概述
通过 Modbus RTU 协议采集 18 路电压数据，提供实时波形显示、数据存储和事件监控。

#### 操作步骤
1. **连接设备**：选择串口号和波特率，点击"连接"按钮
2. **开始采样**：点击"开始采样"开始数据采集
3. **查看波形**：左下区域显示 18 通道实时波形图
4. **查看数据**：右下区域表格显示各通道电压值

#### 工具栏按钮
- **阈值设置**：为每个通道设置超阈值触发电压
- **通道名称配置**：自定义 18 个通道的显示名称
- **事件显示方式**：切换事件列表的显示模式
- **图表显示模式**：切换波形图显示模式
- **切换时间格式**：切换时间戳显示格式
- **导出数据**：导出采集数据到文件
- **关于**：软件版本信息

#### 数据存储
采集的电压数据自动存入 `voltage_data.db` 数据库，包含 18 个通道的电压值和时间戳。

---

### 4.2 NTC 模拟器（第一个标签页）

#### 功能概述
模拟 6 路 NTC 热敏电阻的温度-电压转换，通过 HK 串口模块输出对应电压。

#### 操作步骤
1. **选择串口**：下拉框选择目标 COM 口，填写 Modbus 地址（默认 1）
2. **打开串口**：点击"打开串口"，成功后按钮变为"已连接"
3. **设置通道参数**：为每个通道配置：
   - **通道名**：双击名称框编辑
   - **NTC 曲线**：下拉选择（对应 NTC.xlsx 中的工作表名）
   - **上拉电阻**：单位 kΩ，影响电压输出曲线
   - **温度设定**：直接在输入框输入或拖动下方滑块
   - **上限/下限**：温度变化的上限和下限值（°C）
   - **斜率**：温度变化速度（°C/min），正值升温，负值降温
   - **自动模式**：勾选后通道自动按斜率和限温上升/下降
4. **暂停/恢复**：暂停所有通道的温度更新（与引擎暂停独立）
5. **复位**：将所有通道恢复到 25°C
6. **加载配置**：重新加载 NTC.json 配置文件

#### 通道颜色标识
6 个通道使用不同背景色标识：蓝色、橙色、绿色、黄色、紫色、粉色。

#### NTC 曲线说明
NTC 热敏电阻的阻值-温度曲线存储在 `NTC.xlsx` 文件中。每个工作表代表一条曲线，工作表名即为曲线名称。

文件格式要求：
- 第一列：温度（°C），整数
- 第二列：对应阻值（kΩ），浮点数
- 温度范围建议覆盖 -50°C 到 150°C

默认使用 B=3950 标准 NTC 曲线（R25=10kΩ），电压输出范围约 0.1V ~ 4.9V。

#### 配置文件
通道参数自动保存到 `NTC.json`，下次启动时自动恢复。

---

### 4.3 通道状态（第三个标签页）

#### 功能概述
将通道绑定配置与实时状态看板合并到同一页面，方便在一个视图中完成配置和监控。

#### 上半部分：通道绑定
为 6 个 NTC 通道与 18 个电压采集通道配置对应关系：
1. 在下拉框中为每个 NTC CH1~CH6 选择对应的电压通道（1~18）
2. 设置**阈值电压**：超阈值时引擎触发 NTC 斜坡（在引擎 config.json 中配置）
3. 点击"保存配置"存入数据库
4. 支持**导出模板** / **导入模板**（JSON 文件）

#### 下半部分：状态看板
实时显示 6 个通道的运行状态，包括：

| 列名 | 说明 |
|------|------|
| 通道名 | 自定义通道名称 |
| 设温(°C) | 目标设定温度 |
| 实际V | 实际采集电压值 |
| 斜率 | 当前温度变化斜率 |
| 限幅 | 是否达到温度上下限 |
| 自动 | 是否启用自动模式 |
| 触发 | 是否处于阈值触发状态 |

#### 颜色指示
- **白色**：正常状态
- **黄色**：电压接近阈值（预警）
- **红色**：电压超过阈值（触发中）

#### 事件日志
事件不再持久化到数据库，而是通过 `print()` 输出到运行终端的控制台窗口。事件类型包括：
- `THRESHOLD_TRIGGER`：电压超阈值，触发 NTC 斜坡
- `THRESHOLD_RECOVER`：电压回落阈值以下，停止斜坡
- `HW_FAULT`：大偏差硬件故障（>0.5V）
- `AUTO_STOP`：上位机离线自动停止

## 5. 引擎工作原理

### 5.1 控制循环

引擎以 **100ms 间隔**运行控制循环，每周期执行：

```
1. 存活检测 → 检查 NTC 模拟器和电压采集上位机是否在线
        ↓（掉线则自动停止）
2. 读取数据 → 从数据库和 JSON 文件读取当前电压和 NTC 状态
        ↓
3. 阈值检查 → 对每个通道比较实际电压与设定阈值
        ↓
4. 触发/恢复：
   ├─ 电压 > 阈值 且 未触发 → 写入 NTC 斜坡命令，输出 THRESHOLD_TRIGGER
   ├─ 电压 > 阈值 且 已触发 → 保持斜坡不重复触发，始终输出 THRESHOLD_TRIGGER
   └─ 电压 ≤ 阈值 且 已触发 → 连续低于阈值达到 recover_cycles 次后恢复
        清除斜坡命令，输出 THRESHOLD_RECOVER
        ↓
5. 写状态快照 → 更新状态看板
```

### 5.2 阈值触发机制

控制策略为**开环阈值触发**：

- **触发条件**：实际电压 > 设定阈值，且该通道当前未处于触发状态
- **触发动作**：通过 `NTCJsonWriter.write_cmd()` 向对应 NTC 通道写入预设斜坡参数（斜率、目标温度），NTC 模拟器读取后自动执行升温
- **保持条件**：电压持续超阈值时，斜坡保持运行，不重复触发
- **恢复条件**：实际电压 ≤ 设定阈值，且连续满足 `recover_cycles` 次（默认 5 次 = 0.5 秒）
- **恢复动作**：调用 `NTCJsonWriter.clear_commands()` 清除该通道的外部命令，NTC 模拟器停止自动升温
- **故障上报**：当电压偏差 > 0.5V 时，记录 HW_FAULT 事件（不触发斜坡，避免异常动作）

### 6.3 数据流详解

#### 6.3.1 电压采集数据流

```
DAM-3058 采集模块 (18通道)
    │
    │ Modbus RTU (9600/8N1)
    ▼
acquisition_tab.py (AcquisitionTab)
    │  采集线程: 每 100ms 轮询读取 18 路保持寄存器
    │  → 电压 = 寄存器值 / 1000
    │
    ├──→ 数据队列 (deque maxlen=1000)
    │       │  UI 更新定时器: 每 10ms 追加到波形图
    │       │  存储线程: 每 100ms 批量写入 SQLite
    │       ▼
    │   voltage_data.db / voltage_data 表
    │
    └──→ 事件检测线程: 超阈值比较 → 记录事件记录
```

#### 6.3.2 NTC 温度-电压模拟数据流

```
NTC.xlsx (电阻-温度曲线)
    │  读入 pandas DataFrame
    ▼
NTCConverter
    │  R(T) = lookup(curve, temp)
    │  V_out = 5.0 * R(T) / (R_pullup + R(T))
    │  缓存: voltage_cache[-50..150°C] 预先计算
    │
    └── NTC.json
         │  引擎写入 target_temp / target_slope
         ▼
    NTC 模拟器线程
         │  读取 NTC.json → 更新 NTCConverter → 计算输出电压
         │  通过 HK 串口模块 (Modbus) 输出模拟电压
         ▼
    DAM-3058 采集该电压 → 形成控制回路
```

#### 6.3.3 联动控制数据流（开环阈值触发）

```
┌──────── 联动控制周期 (100ms) ────────────────────────┐
│                                                        │
│  ① 存活检测                                            │
│     NTC.json._heartbeat 时间戳 vs 当前时间             │
│     voltage_data.db 最新记录时间戳 vs 当前时间          │
│     → 任一超时(>10s)则 AUTO_STOP                       │
│                                                        │
│  ② 读取当前状态                                        │
│     电压值 ← voltage_data.db 最新一行 (18 通道)        │
│     NTC 状态 ← NTC.json (temps, curves, slopes, ...)   │
│     阈值配置 ← config.json                             │
│                                                        │
│  ③ 逐通道阈值检查 (6个 NTC 通道)                       │
│     实际电压 vs 设定阈值                               │
│                                                        │
│  ④ 触发/恢复判断                                       │
│     ├─ 电压 > 阈值 且 未触发 → THRESHOLD_TRIGGER      │
│     │    写入斜坡命令到 NTC.json                       │

│                                                        │
│  ⑤ 状态快照写入 link_run_state 表                      │
│     推送 UI (SummaryTab 实时刷新)                      │
│                                                        │
└────────────────────────────────────────────────────────┘
```

```

5. 运行中
   ├── 状态看板实时刷新 6 通道状态
   ├── 事件日志记录触发/恢复/故障事件
   ├── 可操作:
   │   ├── 暂停/恢复引擎
   │   ├── 调整 NTC 参数 (温度/斜率/限幅)
   │   ├── 切换通道自动模式
   │   └── 手动修改阈值
   └── 自动保护:
       ├── 电压超阈值自动触发 NTC 斜坡

6. 结束运行
   ├── 点击"停止" → 复位所有输出
   ├── 关闭窗口 → 自动保存配置
   └── 日志/flush 写入数据库
```

## 7. 配置文件说明

### 7.1 NTC.xlsx
NTC 热敏电阻曲线数据。工作表名为曲线名称，每个工作表包含两列：温度(°C) 和 阻值(kΩ)。

### 7.2 NTC.json
NTC 模拟器配置文件，引擎通过此文件与 NTC 模拟器交互。关键字段：

```json
{
  "ntc_temps":       [25.0, 25.0, ...],  // 6通道当前温度
  "ntc_curves":      ["10k", "10k", ...], // 6通道曲线名称
  "slope":           [5.0, 5.0, ...],     // 6通道斜率(°C/min)
  "auto_run":        [false, ...],        // 6通道自动模式
  "limit_temp_high": [100, ...],           // 6通道温度上限
  "limit_temp_low":  [-50, ...],           // 6通道温度下限
  "pullup_resistors":[10.0, ...],          // 6通道上拉电阻(kΩ)
  "_external_command":{},                  // 引擎命令区
  "_heartbeat":      {...},               // 心跳时间戳
  "_mode":           "standalone"         // 运行模式
}
```

### 7.3 auto_coordinator/config.json
引擎运行参数：

```json
{
  "control_interval_ms": 100,       // 控制周期(ms)
  "alive_timeout_s": 10.0,          // 上位机存活超时(s)
  "state_retention_days": 7,        // 运行状态数据保留天数
  "log_level": "INFO"               // 日志级别
}
```

### 7.4 voltage_data.db

## 9. 常见问题

### Q: 启动引擎时报"NTC.py 或 1.py 离线"
A: 需要先在界面的"电压采集"标签页连接设备并开始采样，在"NTC模拟"标签页打开串口。两个上位机都处于运行状态后，引擎才能启动。

### Q: 启动失败显示"未配置通道绑定"
A: 进入"通道状态"标签页，在上半部分的通道绑定区域为每个 NTC 通道选择对应电压通道，保存配置后再启动。

### Q: 电压输出值偏差过大
A: 检查：
1. 通道绑定是否正确
2. NTC 曲线数据是否与实际热敏电阻匹配
3. 上拉电阻值是否与硬件一致

### Q: 温度变化不响应
A: 确认 NTC 模拟器串口已打开，且串口号与硬件一致。

### Q: 如何更换 NTC 曲线
A: 用 Excel 编辑 `NTC.xlsx`，添加新的工作表（曲线名自定），然后在 NTC 模拟界面下拉切换曲线。
