# 集成测试计划

## 目的
验证联动调度引擎在实际运行场景中的端到端行为。

## 前置条件
- NTC.py 已启动并连接硬件
- 1.py 已启动并采集数据
- voltage_data.db 已存在且有数据
- 已配置通道绑定和联动规则

---

## 场景一：冰箱标准测试

### 步骤
1. 打开 AutoCoordinator 应用
2. 在「通道绑定」页配置：
   - CH1 → CH11, 容差 0.05V, 补偿开
   - CH2 → CH12, 容差 0.05V, 补偿开
   - CH3 → CH13, 容差 0.08V, 补偿开
3. 在「联动规则」页添加规则：
   - CH1 触发 → CH2 联动, 系数 0.4
4. 切换至「状态看板」页
5. 点击「启动」

### 验收检查点
- [ ] 状态栏显示 NTC 和 1.py 均在线
- [ ] 引擎状态变为"运行中"
- [ ] 状态看板刷新频率 ≥ 5Hz
- [ ] CH1 目标温度逐步变化 (斜率 1°C/min)
- [ ] CH2 目标温度逐步变化 (斜率 -2°C/min)
- [ ] CH3 目标温度逐步变化 (斜率 3°C/min)
- [ ] 偏差在容差内时不触发补偿
- [ ] 偏差超过容差时触发补偿 (事件日志可见)
- [ ] CH1 超阈值时 CH2 斜率降低
- [ ] 温度到达限幅点时停止推进, 显示"限幅"
- [ ] 点击停止后输出复位到 25°C

---

## 场景二：异常模拟

### 步骤
1. 启动联动引擎
2. 关闭 NTC.py

### 验收检查点
- [ ] 引擎自动停止
- [ ] 事件日志记录 AUTO_STOP 事件
- [ ] 状态栏 NTC 状态变为离线
- [ ] 停止按钮可用, 启动按钮可用

### 步骤（续）
3. 关闭 1.py

### 验收检查点
- [ ] 引擎自动停止
- [ ] 事件日志记录 AUTO_STOP 事件

---

## 场景三：24h 稳定性

### 步骤
1. 配置完整的 6 通道绑定+联动规则
2. 启动联动引擎
3. 保持运行 24 小时

### 验收检查点
- [ ] 无崩溃/闪退
- [ ] 内存占用稳定 (无持续增长)
- [ ] link_run_state 表数据持续写入
- [ ] link_events 表事件记录完整
- [ ] 偏差补偿不出现持续振荡
- [ ] 斜率降级后能正常恢复
